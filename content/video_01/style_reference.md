# 视频风格参考：不出镜技术视频

## 参考到的传播特征

我看了几类相邻内容：LeRobot/具身智能实操、Python 项目实战、B 站 AI 技术教学。

可以借鉴的点：

- 标题要有明确结果，不要只写“教程”。例如“无脑版”“全过程开源”“无需配环境”这类表达，在 LeRobot 相邻视频里比纯概念标题更容易被点开。
- 开头 10 秒先给结果图，不要先解释背景。
- 口播不要像课程广告，最好像“我刚踩过坑，给你一条省事路线”。
- 画面不要全是 PPT，至少穿插 GitHub、Colab、代码运行和结果图。
- 结尾不要喊太重的“一键三连”，更适合说“代码我放 GitHub 了，想试可以直接跑”。

## 声音选择

首选方向：

```text
年轻男声 / 稍低语速 / 像技术区 UP 主解释问题
```

不建议：

- 新闻播报腔
- 培训机构讲师腔
- 甜美女声
- 过快语速
- 每句话都很完整、很工整的 AI 稿件

如果用免费或可脚本化方案：

```text
zh-CN-YunxiNeural
rate: -10%
pitch: -2Hz 到 -4Hz
```

如果用剪映/CapCut：

```text
选“青年男声 / 知识讲解 / 自然讲述”一类，不选新闻、广告、带货、情绪高昂的声音。
```

如果追求最不像 AI：

```text
自己录音，不出镜，只录声音。
```

不露脸并不等于不能真人配音。用手机录一遍，再做降噪和轻微压缩，通常比任何 TTS 都更像真人。

## 画面节奏

建议 2 分 30 秒到 3 分钟：

- 0-8 秒：直接上结果图，告诉观众“不用硬件也能做”
- 8-25 秒：说明为什么先别买机械臂
- 25-60 秒：打开 GitHub/Colab，跑一遍
- 60-120 秒：解释三张图
- 120-155 秒：说它怎么写进简历
- 155-180 秒：给 GitHub 链接和下一期预告

## 这版视频的调性

一句话：

```text
不是“我来教你一个课程”，而是“我把第一个项目的坑先踩了一遍，你可以直接从这里开始”。
```

口播关键词：

- “先别急着买”
- “我更建议你先跑这个”
- “这个项目很小”
- “它不酷炫，但很适合当第一步”
- “代码我放 GitHub 了”
- “能跑通再说下一步”
