# 计划评估与优化建议

## 结论

这条路线适合当前约束：一个人、业余时间、少售后、无硬件、不做企业销售。最关键的判断是正确的：不要做“LeRobot 教程”，而要做“零硬件具身智能求职项目包”。

## 为什么合适

1. 市场教育已经存在  
   Hugging Face、LeRobot、Datawhale、B 站/CSDN 教程都在帮助用户认识具身智能，所以你不需要从零教育市场。

2. 交付形态仍有空位  
   免费教程通常解决“知道怎么做”，但不一定解决“跑通、整理、写进简历、讲给面试官听”。这个差距正好适合项目包。

3. 零硬件降低决策成本  
   初学者买机械臂之前会犹豫，零硬件项目包更容易让他们先付一个小额费用验证方向。

4. 可连续产品化  
   数据集可视化、BC baseline、VLA 推理评测可以形成自然递进，适合做 3 个小产品，而不是一次性大课。

## 主要风险

1. 环境安装仍然是最大售后来源  
   LeRobot 依赖 Python、视频解码、PyTorch、ffmpeg，不同系统会产生大量错误。

2. 免费内容竞争强  
   如果只讲 LeRobot API，会被官方文档和免费教程替代。

3. 渠道节奏可能过载  
   同时做 B 站、知乎、GitHub、小红书、CSDN/掘金，对业余时间很重。

4. 安全叙事要谨慎  
   第一版应避开 async inference、gRPC、policy server、未知 pickle/model 文件，只做本地可审计脚本。

   当前免费版固定使用 PyPI 可安装的 `lerobot==0.4.4`，并且只覆盖 dataset visualization，不触碰远程推理服务和真实机器人控制链路。

5. 付费边界要清楚  
   免费版太弱无法获客，免费版太强会削弱付费动机。建议免费版给“能跑的结果”，付费版卖“报告、质量分析、简历、面试、FAQ 和节省时间”。

## 优化建议

### 1. 第一版改成 Colab-first

本地环境售后最重。建议付费版优先提供：

- Colab notebook
- Linux / WSL 指南
- macOS 指南
- Windows 原生仅作为非优先支持

免费版保留本地脚本，但销售页强调 Colab 版可以降低安装门槛。

### 2. 前两周先压缩渠道

第 1-2 周只做：

- GitHub
- B 站
- 小报童销售页草稿

知乎、CSDN、掘金、小红书从第 3 周开始复用内容，不要一开始铺太宽。

### 3. 第一屏直接展示结果

README、B 站开头、小报童销售页都要先展示：

- episode 首帧
- action time series
- action distribution
- 一段简历描述

不要先讲背景知识。

### 4. 售后机制前置

必须写清楚：

- 不含一对一环境调试
- FAQ 每月集中更新
- 先运行 `python check_env.py`
- 推荐 Colab / WSL，不优先支持 Windows 原生

### 5. 价格验证优先

第一版早鸟价 49 元合理。比起追求高客单价，第一阶段更应该验证：

- 是否有人愿意买
- 买的人是谁
- 他们为“代码”买单，还是为“简历/报告/面试”买单

### 6. 不要过早承诺 VLA 大项目

项目 03 可以作为路线图，但不要在项目 01 销售页承诺太多未来内容。保持第一版承诺窄而稳。

## 调整后的 MVP

第一周只交付：

- 免费 GitHub 仓库
- 1 个能跑的 notebook
- 1 个 3 分钟 demo 视频
- 1 篇 README
- 1 个早鸟销售页草稿

第二周再补：

- HTML 报告
- FAQ
- 简历模板
- 面试问答
- 付费 zip
